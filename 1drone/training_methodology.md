# Mathematical Framework for Differentiable STL Policy Training

This document details the training methodology implemented in `training/trainer.py`. The approach utilizes **Direct Policy Search via Differentiable Simulation**. Unlike standard Reinforcement Learning (e.g., PPO, REINFORCE) which treats the environment as a black box, this method exploits the differentiability of the system dynamics and the STL robustness semantics to backpropagate gradients directly from the specification to the policy parameters.

## 1. System Dynamics

The environment is modeled as a discrete-time dynamical system. Let \(\mathbf{x}_t \in \mathbb{R}^{N \times F}\) denote the state of all \(N\) nodes (drones, bases, grid points) at time \(t\), where \(F\) is the number of features.

The transition function \(f\) is defined in `drone/dynamics.py`:
\[
\mathbf{x}_{t+1} = f(\mathbf{x}_t, \mathbf{u}_t)
\]

Where \(\mathbf{u}_t = \pi_\theta(\mathbf{x}_t)\) are the actions generated by the policy.

### 1.1 Differentiability
Crucially, \(f\) is implemented using PyTorch operations that preserve the computational graph (e.g., `torch.cat`, `torch.sigmoid`, arithmetic operations). This allows the computation of the Jacobian \(\frac{\partial \mathbf{x}_{t+1}}{\partial \mathbf{u}_t}\) and \(\frac{\partial \mathbf{x}_{t+1}}{\partial \mathbf{x}_t}\), enabling Backpropagation Through Time (BPTT).

**Key Dynamics:**
1.  **Motion:** Momentum-based updates with clamping to grid boundaries.
2.  **Battery:**
    \[
b_{t+1} = \text{clamp}(b_t - \delta_{dis} + \delta_{chg} \cdot \sigma(\gamma (d_{thresh} - d_{base})), 0, 1)
\]
    Where \(\sigma\) is the sigmoid function, making the charging logic differentiable compared to a hard if-then-else threshold.

## 2. Policy Architecture

The policy \(\pi_\theta\) is a neural network parameterized by \(\theta\).

\[
\mathbf{u}_t = \text{tanh}(\text{MLP}_\theta(\text{flatten}(\mathbf{x}_t))) \cdot C_{act}
\]

*   **Input:** Flattened state vector containing drone and base features.
*   **Output:** Continuous velocity/acceleration commands in \(\mathbb{R}^{N_{drones} \times 2}\).
*   **Activation:** `Tanh` ensures actions are bounded within \([-1, 1]\), scaled by \(C_{act}\) (`action_clip`).

## 3. Signal Temporal Logic (STL) and Smooth Robustness

The objective is to satisfy a spatial-temporal specification \(\Phi\). We utilize the **Quantitative Semantics** (Robustness) of STL, denoted as \(\rho(\Phi, \mathbf{x}_{0:T})\).

*   \(\rho > 0 \implies \mathbf{x}_{0:T} \models \Phi\)
*   \(\rho < 0 \implies \mathbf{x}_{0:T} \not\models \Phi\)

### 3.1 Smooth Approximations
Standard STL robustness relies on \(\min\) and \(\max\) operators, which have sparse gradients (gradient is 1 for the extremum, 0 elsewhere). To facilitate training, we use smooth approximations controlled by a temperature parameter \(\beta\) (implemented in `stl/base.py` and `stl/temporal.py`).

**Smooth Min (LogSumExp):**
\[
\widetilde{\min}_\beta(v_1, \dots, v_k) = -\frac{1}{\beta} \ln \left( \sum_{i=1}^k e^{-\beta v_i} \right)
\]

**Smooth Max:**
\[
\widetilde{\max}_\beta(v_1, \dots, v_k) = \frac{1}{\beta} \ln \left( \sum_{i=1}^k e^{\beta v_i} \right)
\]

As \(\beta \to \infty\), these converge to the true hard operators. During training, \(\beta\) is annealed from a low value (soft, dense gradients) to a high value (hard, accurate logic).

## 4. Loss Function

The total loss function \(\mathcal{L}(\theta)\) minimizes the negative robustness while regularizing actions and stabilizing training.

\[
\mathcal{L}(\theta) = \mathcal{L}_{rob} + \lambda_{reg} \mathcal{L}_{reg} + \lambda_{drift} \mathcal{L}_{drift}
\]

### 4.1 Robustness Loss (\(\mathcal{L}_{rob}\))
We aim to maximize the robustness signal over the generated trajectory \(\tau = (\mathbf{x}_0, \dots, \mathbf{x}_T)\).

\[
\mathcal{L}_{rob} = - \mathbb{E}_{\tau \sim \pi_\theta} \left[ \text{mean}_{env}(\rho(\Phi, \tau)) \right]
\]

### 4.2 Action Regularization (\(\mathcal{L}_{reg}\))
To encourage energy efficiency and smooth control:

\[
\mathcal{L}_{reg} = \frac{1}{T} \sum_{t=0}^{T} \| \mathbf{u}_t \|^2_2
\]

### 4.3 Asymmetric Drift Loss (\(\mathcal{L}_{drift}\))
Implemented in `_asymmetric_drift_loss`, this component prevents "catastrophic forgetting" or trajectory collapse during updates. It penalizes the policy if the state trajectory deviates significantly from the previous iteration's trajectory, **but only if the performance (battery cost) degrades**.

Let \(\tau^{(k)}\) be the trajectory at iteration \(k\), and \(\tau^{(k-1)}\) be the detached trajectory from the previous iteration.

1.  **Cost Metric:** \(C(\mathbf{x}) = - \text{mean}(\text{battery})\). (Higher cost = lower battery).
2.  **Valve:**
    \[
\delta = \text{ReLU}(C(\tau^{(k)}) - C(\tau^{(k-1)}))
\]
    \(\delta > 0\) only if the new trajectory has worse battery usage than the old one.
3.  **Penalty:**
    \[
\mathcal{L}_{drift} = \frac{1}{T} \sum_{t=0}^T \delta_t \cdot \| \mathbf{x}_t^{(k)} - \mathbf{x}_t^{(k-1)} \|^2_2
\]

This allows the policy to change the trajectory freely as long as it improves (or maintains) battery efficiency, but restricts movement if it leads to worse outcomes.

## 5. Training Algorithm

The training loop (`train_policy_gradient`) proceeds as follows:

1.  **Initialization:**
    *   Initialize policy \(\pi_\theta\).
    *   Set annealing schedules for \(\beta\) (smoothness) and \(\sigma\) (exploration noise).

2.  **Iteration Loop (\(k = 1 \dots K\)):**
    *   **Update Hyperparameters:**
        *   \(\beta_k \leftarrow \text{Anneal}(\beta_{start}, \beta_{end}, k)\)
        *   \(\sigma_k \leftarrow \text{Anneal}(\sigma_{start}, \sigma_{end}, k)\)
        *   Update STL operators globally via `Node.set_smooth_beta(\beta_k)`.

    *   **Rollout (Forward Pass):**
        *   Sample initial states \(\mathbf{x}_0\).
        *   Simulate \(T\) steps:
            \[
\mathbf{u}_t = \pi_\theta(\mathbf{x}_t) + \epsilon_t, \quad \epsilon_t \sim \mathcal{N}(0, \sigma_k^2)
\]
            \[
\mathbf{x}_{t+1} = f(\mathbf{x}_t, \mathbf{u}_t)
\]
        *   Store trajectory \(\tau\).

    *   **Evaluation:**
        *   Compute Robustness map: \(R = \text{Spec}(\tau)\).
        *   Extract min robustness over time per environment.

    *   **Loss Computation:**
        *   Calculate \(\mathcal{L}_{rob}\), \(\mathcal{L}_{reg}\).
        *   If \(k > 0\), calculate \(\mathcal{L}_{drift}\) using cached states from \(k-1\).

    *   **Optimization (Backward Pass):**
        *   Compute gradients \(\nabla_\theta \mathcal{L}\) via BPTT.
        *   Clip gradients to norm \(C_{grad}\).
        *   Update \(\theta\) using Adam optimizer.

    *   **Cache:**
        *   Detach current trajectory \(\tau\) and store as \(\tau^{(k-1)}\) for the next drift calculation.

## 6. Summary of Strategies

| Strategy | Implementation | Purpose |
| :--- | :--- | :--- |
| **Differentiable Physics** | `drone/dynamics.py` | Allows gradients to flow from future states back to actions. |
| **Smooth STL** | `stl/` (LogSumExp) | Provides dense gradients for logic operators (\(\min, \max\)). |
| **Curriculum (Beta)** | `smooth_beta_anneal` | Starts with easy optimization landscape, converges to exact logic. |
| **Exploration Decay** | `exploration_std` | High noise initially for exploration, reduced for fine-tuning. |
| **Asymmetric Drift** | `_asymmetric_drift_loss` | Stabilizes training by penalizing regressive trajectory changes. |
| **Gradient Clipping** | `clip_grad_norm_` | Prevents exploding gradients common in BPTT. |
